=== 사용자 기능 흐름 ===
사용자가 웹 애플리케이션에 접속하면 `전체화면-section`이 표시되며, 이 화면은 `AI채팅-display` 영역, `엑셀시트-section`, 그리고 `확인리스트-display`를 포함합니다.

사용자는 먼저 `업로드버튼-section`을 클릭하여 파일 업로드를 시작합니다. 이 행동은 `임시저장 묻기-modal`을 활성화하여 파일 탐색기를 열고 사용자에게 엑셀 파일 선택을 요청합니다. 사용자가 파일을 선택하고 저장을 확정하면, 해당 엑셀 파일은 `임시폴더-folder`에 저장됩니다. `임시폴더 저장 인식 기능-function`은 `임시폴더-folder`의 변경을 감지하여 `임시저장-display` 내의 `문서 리스트-section`를 업데이트하고, 이 업데이트된 리스트는 `임시저장-display`에 표시됩니다.

`임시저장-display`에서 사용자는 특정 `파일 제목-section`을 클릭하여 해당 파일의 내용을 확인할 수 있습니다. `파일 제목-section`이 클릭되면, 관련된 데이터가 `엑셀시트-section`에 엑셀 형식으로 로드되어 표시됩니다.

임시 저장된 파일을 최종본으로 저장하려면, 사용자는 `문서 리스트-section` 내의 `저장-section` 버튼을 클릭합니다. 이 클릭은 엑셀 데이터를 `가변적 데이터-data`로 전송하며, `가변적 데이터-data`는 엑셀 헤더를 기반으로 필드명을 동적으로 구성하고 데이터를 SQL 형식으로 처리하여 `파일내용 폴더-folder`에 저장합니다. 동시에 `파일 관리-data`는 파일의 타임스태프와 파일이름 같은 메타데이터를 관리하고 `파일관리 폴더-folder`에 저장합니다. `파일 관리-data`에서 처리된 데이터는 `파일-function`으로 전달되며, `파일-function`은 이 데이터를 `문서 리스트(1)-section`의 양식에 맞춰 `최종본-display`에 최종본 파일로 업로드합니다. `문서 리스트(1)-section`은 이 업데이트된 리스트를 `최종본-display`에 표시합니다. 만약 임시 파일을 더 이상 필요로 하지 않는다면, 사용자는 `문서 리스트-section` 내의 `삭제-section` 버튼을 클릭하여 `임시저장-display`에서 해당 파일을 제거할 수 있습니다.

`최종본-display`에서는 `문서 리스트(1)-section`에 있는 `파일 제목(1)-section`을 클릭하여 해당 파일의 내용을 `엑셀시트-section`에 표시할 수 있습니다. `엑셀시트-section`에 표시된 내용을 실제 엑셀 파일로 영구 저장하려면, 사용자는 `저장버튼-section`을 클릭하며, 이 행동은 엑셀 데이터를 `엑셀파일 저장 폴더-folder`에 저장합니다. 또한, `문서 리스트(1)-section` 내의 `저장(1)-section`도 `엑셀파일 저장 폴더-folder`와 연결되어 있어, 특정 최종본 문서를 영구 저장하는 기능을 수행할 수 있습니다. 최종본 파일을 삭제하려면, 사용자는 `문서 리스트(1)-section` 내의 `삭제(1)-section` 버튼을 클릭하여 `최종본-display`에서 해당 파일을 제거합니다.

한편, 사용자는 `채팅창-section`에 프롬프트를 입력하여 AI와 상호작용할 수 있습니다. 입력된 프롬프트는 AI에 의해 처리되고, AI의 해석 또는 생성된 내용은 `AI채팅-display` 내의 `AI채팅창 화면-section`에 표시됩니다. AI의 결과물은 `엑셀시트-section`으로 전송되어 엑셀 데이터를 수정하거나 새로운 데이터를 생성할 수 있습니다. `엑셀시트-section`의 데이터는 `확인리스트-display`로도 전달되어 사용자가 검토할 수 있습니다.

`AI채팅창 화면-section`에 표시된 내용(및 `엑셀시트-section`의 데이터)을 기반으로, 사용자는 `AI 마법사-modal`을 활성화할 수 있습니다. `AI 마법사-modal`은 엑셀 파일의 내용을 활용하여 `AI슬라이드-display`에 PPT 슬라이드를 생성하고, `AI결재-display`에 그룹웨어 결재 문서를 작성합니다.

=== UI 요소 상호작용 ===
### UI 상호작용

사용자가 `업로드버튼-section`을 클릭하면 파일 탐색기가 열리며, 엑셀 파일만 업로드할 수 있습니다. 파일 선택 후 `임시저장 묻기-modal`이 나타나 파일 저장 여부를 다시 확인하며, 여기서 '저장' 버튼을 누르면 파일이 `임시폴더-folder`에 저장됩니다. `임시폴더-folder`에 파일이 저장되거나 추가되면 `임시폴더 저장 인식 기능-function`이 이를 감지하여 `임시저장-display`에 `문서 리스트-section` 형태로 임시 저장된 파일 목록을 업데이트하여 보여줍니다. `임시저장-display` 내 `문서 리스트-section`에 있는 `파일 제목-section`을 클릭하면 해당 파일의 데이터가 `엑셀시트-section`에 엑셀 형식으로 표시됩니다. 또한, `문서 리스트-section` 내 `삭제-section`을 클릭하면 해당 파일이 임시 저장 리스트에서 삭제됩니다.

한편, 사용자가 `채팅창-section`에 프롬프트를 입력하면, 이 프롬프트는 `AI채팅창 화면-section`으로 전달되어 AI가 내용을 해석하고, 그 해석 결과물은 `엑셀시트-section`에 엑셀 형식으로 띄워집니다. `엑셀시트-section`에 표시된 데이터는 `확인리스트-display`에도 연동되어 사용자에게 추가 확인을 제공합니다. `AI채팅창 화면-section`의 결과물은 `AI 마법사-modal`을 통해 엑셀 파일 내용을 바탕으로 PPT 슬라이드를 작성하는 데 활용될 수 있으며, 이 과정에서 `AI슬라이드-display`에 슬라이드가 생성되고 `AI결재-display`에는 그룹웨어 결재 문서가 작성됩니다.

`엑셀시트-section`에서 작업된 내용을 `저장버튼-section`을 클릭하면 실제 엑셀 파일로 `엑셀파일 저장 폴더-folder`에 저장됩니다. `임시저장-display` 내 `문서 리스트-section`에서 `저장-section`을 클릭하면 해당 파일이 `최종본-display`의 `문서 리스트(1)-section`으로 이동합니다. `파일 관리-data`의 데이터가 변동되면 `파일-function`이 작동하여 `문서 리스트(1)-section`의 양식에 따라 `최종본-display`에 최종본 파일 목록을 보여줍니다. `최종본-display` 내 `문서 리스트(1)-section`에 있는 `파일 제목(1)-section`을 클릭하면 관련된 데이터가 `엑셀시트-section`에 엑셀 형식으로 표시됩니다. 마지막으로, `문서 리스트(1)-section` 내 `삭제(1)-section`을 클릭하면 해당 파일이 최종본 리스트에서 삭제됩니다.

### 데이터 흐름

`업로드버튼-section`을 통해 사용자가 엑셀 파일을 선택하면, `임시저장 묻기-modal`을 거쳐 파일 데이터가 `임시폴더-folder`로 전달됩니다. `임시폴더-folder`에 파일이 저장되면 `임시폴더 저장 인식 기능-function`이 트리거되어, 이 기능은 `문서 리스트-section`을 업데이트하고, 업데이트된 리스트는 `임시저장-display`에 표시됩니다. `문서 리스트-section` 내 `파일 제목-section` 클릭 시, 해당 파일의 데이터가 `엑셀시트-section`으로 로드됩니다.

`채팅창-section`에 입력된 프롬프트 텍스트 데이터는 `AI채팅창 화면-section`으로 전달되어 AI에 의해 해석됩니다. AI가 해석한 결과 데이터는 `엑셀시트-section`으로 전달되어 엑셀 형식으로 시각화되며, 이 데이터는 다시 `확인리스트-display`로 전달되어 사용자에게 추가 확인을 제공합니다. `AI채팅창 화면-section`의 결과 데이터는 `AI 마법사-modal`로 전달되어 PPT 슬라이드 작성에 활용되며, 이는 `AI슬라이드-display`로 출력됩니다. 또한, 엑셀 내용을 바탕으로 그룹웨어 결재 문서가 `AI결재-display`에 작성됩니다.

`엑셀시트-section`에서 작업된 최종 엑셀 데이터는 `저장버튼-section`을 통해 `엑셀파일 저장 폴더-folder`에 실제 엑셀 파일로 저장됩니다. `임시저장-display` 내 `문서 리스트-section`에서 `저장-section`을 클릭하면, 해당 파일의 데이터가 `가변적 데이터-data`와 `파일 관리-data`로 전달됩니다. `가변적 데이터-data`는 업로드되는 엑셀의 헤더인 1행을 따라 필드명을 구성하고 SQL 형식으로 `파일내용 폴더-folder`에 저장하며, `파일 관리-data`는 파일의 타임스탬프와 파일 이름을 포함한 관리 데이터를 `파일관리 폴더-folder`에 저장합니다. `파일 관리-data`의 데이터 변동은 `파일-function`을 트리거하며, `파일-function`은 `문서 리스트(1)-section`을 업데이트하여 `최종본-display`에 최종본 파일 목록을 표시합니다. `문서 리스트(1)-section` 내 `파일 제목(1)-section` 클릭 시, 해당 파일의 데이터가 `엑셀시트-section`으로 로드됩니다.
